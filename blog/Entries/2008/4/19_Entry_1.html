Dead badger (Octane) runs Linux natively

Saturday, April 19, 2008

 

Yes, it only took 3 months (during lunch hours), but Aaron and Peter got the SGI Octane to run Gentoo natively.  The hell that they went through was a wonderful demonstration of “sweat equity” - a heavy combination of research and trial-and-error.   After learning how to prepare the system and compile the necessary software including the kernel (the steps are in my March 15th blog entry), we realized that the latest kernel doesn’t like the R12000 platform and there are no working patches to make it work.


So.......Plan B was to directly image an SGI volume header from a known working MIPS Live CD, extract the kernel from it using dvhtool and copy it across the network for storage.  Next, the Octane would be prepared as normal (a long process), but the known working kernel would be inserted into the SGI volume header instead of compiling the new kernel version.  However, the arc.cf configuration file was custom made for the Live CD and it wouldn’t boot the Gentoo system on the hard disk.  It took quite a bit of trial-and-error to figure out how to confuse the new kernel to boot the Gentoo system on the first SCSI hard disk (/dev/sda1) - that was done this week, and the Octane natively boots Gentoo Linux.

Once the system is booted, you can log into terminals to interact with the BASH shell. Below is a root logon that shows the /proc/cpuinfo file that lists the R12000 CPU:




The system is quite fast - in the next little while, Peter and Aaron will be configuring the system more and installing software to see how fast the Octane performs.  I installed Gentoo a year ago on an Octane and it was unbelievably fast (it smoked our classroom Pentium 4 (3GHz) systems with just Apache benchmarks.  I think it will smoke our Core2Duos that we have now as well.


For a detailed list of what Aaron and Peter did, download this file: Installing Linux on a Dead Badger using Reverse Osmosis, Octane MIPS R12000.doc
