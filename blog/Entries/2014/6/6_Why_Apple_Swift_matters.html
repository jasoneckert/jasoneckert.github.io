Why Apple Swift matters

Friday, June 6, 2014

 

This past week Apple announced a new programming language called Swift, that they’ve created to replace their existing Objective-C language.  For app developers, it was an epic announcement - everyone in our developer faculties talked about it on our calls that week and I received at least a few dozen Swift-related emails each day during the week from my developer friends.


But for everyone else, Apple’s announcement of Swift was just plain boring - they would rather have heard about a new iDevice of some type.   However, non-developers should be as excited about Apple’s new Swift language as developers are, and in this post I’m going to tell you why.


Firstly, a programming language is simply an abstract language that follows some set rules and allows you to interface with a plethora of build-in functions for a specific computing device platform (in this case, the Apple iPhone, iPad, iMac, and so on).  


Secondly, the whole purpose of development is to create apps, and apps can be created easier and faster if the developer toolset used:

• Is easy to work with
• Has more functionality

In recent years, programming languages that use simple syntax and whitespace have been gaining traction as a result of their simplicity.  These languages are easier to write, easier for others to read, and this ultimately means that apps can be made quicker and with fewer human mistakes.  Swift definitely falls into this category when compared with Objective-C.  For example, to create a string of text that can be used within other parts of the program (e.g. to display the words “Hello World“), you had to create an NSString (NeXT Software String) object that held the letters “Hello”, and then append a space and the letters “World” afterwards:


NSString *str = @"Hello,";

str = [str stringByAppendingString:@" World"];


However, in Swift, you could simply create a string variable that held the letters “Hello” and use the concatenation operator (+) to append a space and the letters “World”:


var str = "Hello,";

str += " World"


Which language would you prefer to code in?  I guessing you chose the easier one...


When it comes to functionality, imagine that the programming language you have comes with an easy-to-use built in function that can be used to draw a 3D cube.  You can type a single line of code that uses that function with the parameters that you specify.  Now imagine that your language had no such 3D cube function, but instead has a line drawing function.  You would need to write a tremendous amount of code just to draw the cube using lines that make it appear 3D, and in a way that can be modified if the cube is moved or rotated.  Not fun.  Groups of key functions, like the 3D cube one we talked about, are often called Application Programming Interfaces (APIs), and Apple has made over 4000 new APIs available for Apple devices, including one called Metal, that makes it much easier to create 3D games for iOS! 


In short, by announcing a series of APIs that make it easier to develop apps, and a new programming language that is easier to work with, Apple hasn’t just made the lives of developers easier - they’ve made it easier to create more and better apps that can do more than the apps we have today, as well as encourage more people to enter the app developer profession. And this is why everyone should be excited about Apple’s announcements this past week - expect to see some amazing apps soon!
