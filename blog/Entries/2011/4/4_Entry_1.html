Old school computing

Monday, April 4, 2011

 

Bet you haven’t seen a system like the one in the picture above (or recently at least)....but back in the 1980s and early 1990s, these things were all over the place, especially in universities and scientific/engineering organizations.  


What you are looking at is a VMS (Virtual Memory System) operating system running on a DEC Alpha computer.   I mentioned DEC in a few previous posts, but someone asked me to post a screenshot of what it was like, so here it is.


Back in the old days (the 1980s), this operating system ran all over the place on DEC VAX (Virtual Address eXtension) computers.  They weren’t that flashy, and they had their own unique command line interface that will seem foreign today - for example, here is a nice snippet from Phil Ottewell’s VMS tutorial:


EXAMP :== "SYS$EXAMPLES:"

COPY 'EXAMP'LIBTEST1.FOR VAXLIB1.FOR

COPY 'EXAMP'LIBTEST2.FOR VAXLIB2.FOR

FORT VAXLIB1

FORT VAXLIB2

LIB/CREATE MYLIB VAXLIB2

LIB/LIST/FULL MYLIB

LINK VAXLIB1,MYLIB/LIB

RUN VAXLIB1


You may be able to follow what is happening here - we first copy the files LIBTEST1.FOR and LIBTEST2.FOR from the SYS$EXAMPLES: directory to our current directory and call them VAXLIB1.FOR and VAXLIB2.FOR respectively.  We then compile both FORTRAN files, convert VAXLIB2 into a library called MYLIB, inspect the library contents, link VAXLIB1 with this library and and RUN the resulting EXE file.  Remember that FORTRAN was the only “proven” language in the scientific/engineering community until very recently!


Similarly, files on a VMS system have a mandatory file extension and versioning information (you can have multiple versions of the same file) - for example, PROG.EXE;14 refers to the 14th version of the PROG.EXE file.  You can also make aliases for certain files or directories (e.g. ASSIGN/NOLOG DEV$DISK:[JASON.ECKERT] JASON$DIR:) or copy files between VMS machines easily (e.g. COPY DEV$DISK:[JASON]TCM.TEX PDS$DISK:[BOB]).  You get used to it.


VMS influenced all other operating systems too!  It was the first commercial OS to include integrated networking (DECnet and TCP/IP), support for multiple programming platforms (FORTRAN, C, C++, Ada, BLISS, etc.), distributed network file systems, rights management systems (RMS),  multi-CPU hardware partitioning,  NUMA multiprocessing, and network clustering (where multiple systems across the network share processing, disk storage, print queues, etc.).


Anywho, VMS is still around!  After the VAX days of the 1980s, VMS was used on DEC Alpha servers in the 1990s, until the mid 2000s.  Now it runs on Itanium HP servers - the latest version is 8.4 (June 2010).  That doesn’t mean that you should run out and learn it anytime soon - VMS is only used in a small number of environments - but it is neat to see that it is still around.  I used a lot of VMS boxes in the past, and I liked the system (it was very stable and fast), but I was more of a UNIX nut back then.  I probably used VMS as much as UNIX, but most people religiously chose one platform or the other.  In the late 1980s, if you put a bunch of VMS and UNIX admins in the same room, there would be bloodshed ;-)
