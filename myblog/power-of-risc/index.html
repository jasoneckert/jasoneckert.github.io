The Power of RISC architecture

2022-02-05 +
Reading Time: 3 mins
Section: Blog Posts
Categories: blog
Tags: tech Apple computing UNIX macOS IT

This part of the 1995 Hackers movie illustrates is right on par for the time. After all, by the mid 1990s the Reduced Instruction Set Architecture (RISC) processors were seen as the future of computing by most in the tech industry.

By using a simplified instruction set, computers with RISC processors could perform tasks in a quarter of the time compared to computers with a traditional Complex Instruction Set Architecture (CISC) processor. RISC processors also required less circuitry, which allowed manufacturers to easily add more functionality, such as crypto accelerators, memory management controllers and specialized floating point arithmetic processing units.

Most PCs and Macs of the early 1990s used Intel or Motorola CISC processors. And while RISC processors were available at the time, they were only used in very expensive high-end UNIX workstations. Apple used PowerPC RISC processors in their Macs during the late 1990s, but switched to the Intel CISC processors that have always dominated the market by the mid 2000s. In fact, by the late 2000s, most RISC processor platforms had long died off with the exception of the Acorn RISC Machine (ARM) processor that dominated the low-power device and mobile markets since the 1990s. For example, your smartphone, Google Home, and car navigation system all use ARM processors.

However, there really is no reason why you can’t make a higher-power ARM processor for computing. And this is exactly what many different manufacturers started doing in the late 2010s - Amazon’s Graviton powerful server processor and Apple’s M1 desktop processor are the two most famous examples.

To illustrate the power of RISC architecture, let’s compare two different Mac computers running the same version of macOS. One has an Intel Core i9 CISC processor and 64GB of memory, while the other has the Apple M1 RISC processor and 16GB of memory. This means that I can composite my website on both the Intel Core i9 CISC processor and Apple M1 RISC processor using the same version of macOS and hugo:

The Intel Core i9 took 6126 milliseconds (6 seconds) to generate all 273 pages of website, while the Apple M1 only took 549 milliseconds (half a second). The Apple M1 was over 11 times faster!

Perhaps part of the discrepancy was related to the storage speed. Both systems use solid state storage that is soldered to the motherboard, but their speeds could be different. So I ran the time command to measure the time it takes to create a 1073 MB file (called tstfile) using the dd command:

The storage speed of both systems is nearly identical (0.36 seconds to create the same file), so storage didn’t contribute to the results. While the unified memory in the Apple M1 is closer to the processor than the DDR4 memory in the Intel Core i9, the benefits it provides to hugo are probably a lot less dramatic than for complex/graphical applications. Thus, most of the difference in hugo speed between the Intel Core i9 and Apple M1 is likely due to the difference between RISC and CISC architecture.

Looks like Kate and Dade from Hackers were correct. Go figure.
